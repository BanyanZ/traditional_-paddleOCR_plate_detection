# 传统图像处理 + PaddleOCR 车牌检测识别项目

这是一个用于演示“中国车牌图像检测、提取、颜色判断、字符分割和 OCR 识别”的图像处理项目。项目把传统 OpenCV 图像处理方法和 PaddleOCR 文字识别结合起来，并通过 PyQt5 做成一个简单的桌面可视化程序，用户可以选择一张车牌图片，逐步查看车牌定位、车牌区域提取、车牌颜色识别、字符分割以及最终识别出的车牌号码。

项目适合作为图像处理课程设计、车牌识别流程演示、OpenCV 形态学处理实验或 PaddleOCR 入门应用示例。

## 项目功能

- 读取本地车牌图片，支持 `.jpg`、`.jpeg`、`.png`、`.bmp` 等常见格式。
- 使用 OpenCV 形态学方法从整张图片中定位车牌区域。
- 在原图上用红色矩形框标出检测到的车牌位置。
- 裁剪并放大车牌 ROI，方便后续观察和识别。
- 根据车牌上下边缘区域的 HSV 颜色分布判断车牌背景颜色，支持蓝色、黄色、绿色，无法确定时返回未知。
- 对提取出的车牌区域进行灰度化、OTSU 二值化、噪声过滤和字符轮廓分割。
- 使用 PaddleOCR 对车牌 ROI 进行中文车牌文字识别。
- 使用 PyQt5 提供可视化交互界面，按钮式完成“显示图片、车牌提取、字符分割、车牌识别”等操作。

## 环境要求

推荐环境：

- Python 3.9
- Windows 系统或其他支持 PyQt5 图形界面的系统
- 能够正常安装 PaddlePaddle / PaddleOCR 的 Python 环境

主要依赖：

- `opencv-python`：图像读取、颜色空间转换、形态学处理、轮廓检测、绘制矩形框等。
- `paddlepaddle`：PaddleOCR 的底层深度学习框架。
- `paddleocr`：车牌文字 OCR 识别。
- `numpy`：图像矩阵和数值计算。
- `matplotlib`：图像可视化辅助库。
- `PyQt5`：桌面窗口界面。

## 界面使用说明

程序启动后，会出现一个车牌识别演示窗口。界面左侧显示原图或带红框的检测结果，右侧显示车牌提取结果、车牌颜色、字符分割结果和 OCR 识别结果。

按钮说明：

- `显示图片`：从本地选择一张车牌图片并显示在左侧。
- `车牌提取`：对当前图片进行车牌定位，提取车牌 ROI，并在原图上绘制红色检测框。
- `字符分割`：对已经提取出的车牌区域做字符分割，并在右侧显示分割后的字符条带。
- `车牌识别`：调用 PaddleOCR 对车牌 ROI 进行文字识别，并显示识别出的车牌号码。

建议操作顺序：

```text
显示图片 -> 车牌提取 -> 字符分割 -> 车牌识别
```

## 核心算法流程

### 1. 车牌区域定位

核心函数：

```python
Morph_Distinguish(img)
Morph_Distinguish_with_box(img)
```

处理流程：

1. 将输入图像转为灰度图。
2. 使用矩形结构元素进行顶帽变换，突出较亮的车牌字符和边缘区域。
3. 使用 Sobel 算子提取水平方向边缘特征。
4. 对边缘图进行固定阈值二值化。
5. 通过开运算、闭运算、腐蚀、膨胀等形态学操作连接车牌字符区域，去除孤立噪声。
6. 使用 `cv2.findContours` 查找候选轮廓。
7. 根据车牌常见宽高比和面积条件筛选候选区域。
8. 返回车牌 ROI；增强版本还会返回车牌矩形框坐标 `(x, y, w, h)`。

### 2. 车牌颜色识别

核心函数：

```python
recognize_plate_color(plate_bgr)
```

处理流程：

1. 将车牌 ROI 从 BGR 转换到 HSV 色彩空间。
2. 取车牌上边缘和下边缘区域作为背景色采样区域，减少字符颜色对判断结果的干扰。
3. 分别统计蓝色、黄色、绿色 HSV 范围内的像素数量。
4. 计算各颜色像素占比。
5. 返回占比最高且超过阈值的颜色；否则返回“未知”。

该部分主要用于区分常见蓝牌、黄牌和新能源绿牌。

### 3. 字符分割

核心函数：

```python
segment_characters(plate_bgr)
```

处理流程：

1. 将车牌 ROI 转为灰度图。
2. 使用高斯滤波降低噪声。
3. 使用 OTSU 自动阈值进行二值化。
4. 保证结果为白色字符、黑色背景。
5. 使用形态学开运算进一步去噪。
6. 查找字符轮廓。
7. 根据字符高度、宽度、面积、宽高比过滤噪声点、粘连块和干扰区域。
8. 按字符横坐标从左到右排序。
9. 将每个字符缩放到统一高度，并横向拼接成一条字符分割结果图。

字符分割结果主要用于可视化展示传统图像处理中的分割效果，最终文字识别仍由 PaddleOCR 完成。

### 4. OCR 车牌文字识别

核心代码：

```python
self.ocr = PaddleOCR(use_angle_cls=True, lang="ch")
result = self.ocr.ocr(self.roi_bgr, cls=True)
```

程序使用 PaddleOCR 的中文识别模型对提取出的车牌 ROI 进行识别，并对结果文本做简单清理：

- 去除空格。
- 去除英文句点 `.`。
- 如果未识别到文本，则显示“未识别到”。

## 局限性

- 车牌定位使用传统形态学规则，对光照、角度、遮挡、复杂背景比较敏感。
- 候选区域筛选主要依赖宽高比和面积阈值，面对倾斜严重或尺寸很小的车牌可能检测失败。
- 颜色识别依赖 HSV 阈值和边缘区域采样，遇到反光、污损、曝光异常时可能返回“未知”或错误颜色。
- 字符分割是传统轮廓分割方法，遇到字符粘连、模糊、车牌边框干扰时效果会下降。
- OCR 识别准确率依赖车牌 ROI 的清晰度、裁剪质量和 PaddleOCR 模型表现。

## 常见问题

### 运行时 PaddleOCR 首次启动很慢

这是正常现象。PaddleOCR 第一次运行时会下载检测模型、识别模型和方向分类模型，下载完成后再次运行会快很多。

### 运行时报 PyQt5 相关错误

请确认已经安装 PyQt5：

```bash
pip install PyQt5
```

如果在无图形界面的服务器环境运行，PyQt5 窗口可能无法正常显示，建议在本地桌面环境运行。

### 没有检测到车牌

可能原因包括：

- 图片中的车牌太小或太模糊。
- 车牌倾斜角度较大。
- 背景中存在与车牌形状类似的强干扰区域。
- 当前形态学阈值不适合这张图片。

可以尝试换用 `车牌图像` 目录中的其他样例图片，或调整 Notebook 中的阈值、结构元素大小和候选区域筛选条件。

### 识别结果不准确

可以先观察“车牌提取结果”是否完整、清晰。如果车牌 ROI 裁剪不完整，OCR 结果通常也会受到影响。可通过改进车牌定位参数、增加图像预处理或换用更清晰图片提升效果。
